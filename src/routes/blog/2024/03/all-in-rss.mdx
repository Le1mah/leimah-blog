# All in RSS

嗯……标题夸张了点。本质上是想简单聊聊 RSS 和最近的一点小折腾。

先说 RSS 本身吧。
第一次听说 RSS 这么个概念大概要追溯到十年前，彼时亲戚家的新电脑是最新最酷的 Vista 系统，和自己家里 XP 系统相比，像是两个时代的产物。
虽然是寒假，但表哥并不在家，我便得以随意地尝试各种新鲜的玩意，除了桌面上的小工具之外，我能想起的就是新款 IE 上的橙色小方块了。

可惜那时的我还沉迷于用像素点阵画程序图标、又或者是摆弄新版 PowerPoint 的动效，对 RSS 这样的概念也只是浅尝辄止。

现在回想起来，不光是 RSS, 还有多少事情这样被我错过了呢？只是想想就很后悔。

{/* 事情的转机大体是在去年的年末，似乎是在看 [Jack Franklin 的博客](https://www.jackfranklin.co.uk/blog/) 时，被那样大号的 RSS 三个字母抓住了眼球。
我突然反应过来：似乎自己看的博客里，支持 RSS 的数量比预想的要多很多。

首先尝试的是 Fluent Reader, 似乎是现今少有的免费开源的 RSS 阅读器。

在使用了一段时间后，我发现了一个很严重的问题：如果 WFH 还好，但若是去公司，就不得不考虑同步的问题了。

几经纠结后，我把目光转向了 iPad, 若是 iOS 上有不错的 RSS 阅读器，那么不仅不用担心同步的问题，更是可以在上下班的路上看一看更新了。
 */}

虽然没法一时间捡起所有被我遗忘的点子，但至少 RSS，我是可以现在重新开始的。

在辗转 PC 和 iOS 几个月之后，每天看看 RSS 的更新已然变成了雷打不动的习惯。

它实在是太方便了。

要说先前在浏览器收藏夹之间跳转的样子，像是去报刊亭自己一个个地挑选想要的日报。那么 RSS 就像是报童，早上醒来就可以在家门口拿到一沓当日的新闻。

难怪有人愿意为报童付钱！我也是一样，几乎没有一点犹豫地买了 Reeder。它的体验同样是没有让我失望——无论是午休时，上下班的路上，乃至外出吃饭等位，也可以抽出几分钟略读一下今天的新事情。

### 承：
是不是它也可以被改写成 RSS 呢？
某个夜晚，再一次被 504 折磨搞到心态的我突然灵光乍现。

方案本身非常简单，
群晖 WebStation 已经把 Nginx 和 uwsgi 准备好了。
简单写一个支持 uwsgi 的 Web App（Flask呀、Tornado呀）。
在 NAS 里简单配一下就搞定。

说干就干，下班时趁着网页还在正常工作，先存下了响应的样例——居然是类 XML 的文档……不得不感慨系统运行之稳定……

上一次处理 XML 可以回溯到我的第一份工作。
那时的我也是基于别人的代码上修修改改。
虽然并不是什么难事，但选框架、读文档还是有点麻烦的事情。

正好也是一个用 Copilot 的好时机。
第一个月的免费试用已经到了尾声，却还没怎么用过。

实际的操作也十分简单，写上大体的注释，代码就会大段大段地蹦出来。
我也试过用 GPT 去写，但 Web UI 总是要等到一批结果生成之后才好进行下一次迭代。
这点上 Copilot 的综合开发体验稍强一分（如果有了类似的 GPT 插件，也请务必告诉我，非常感谢~）。

代码整体也没有什么好说的。
API 返回的 XML 文档中有若干 \<record> 实体，其中是每一条通知对应的 HTML 元素。
内容很少，不需要引入别的库来解析，只需要一点正则就能提取出需要的信息。
尔后再同样用 Python 自带的 xml 库转换成 RSS 订阅，作为 API 的输出即可。

### 转：
毕竟是一个非常简单的小工具，第二天下班的时候就趁着还没 504 测试了一波。看起来没有什么异常。只需要再移动到 NAS 上即可。

但预想的 XML 文档并没有如我想像一般出现。而是一个大大的 Internal Server Error……像是当头一棒打得我发懵。
